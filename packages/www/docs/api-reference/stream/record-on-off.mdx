# `PATCH` toggle recording

`PATCH /stream/{id}/record` can only modify a `stream` object. You cannot modify
the `record` value on a `session` object or the historic `stream` objects with a
`parentId` (stream objects representing a single live stream session).

The record value is inherited by all future child `session` objects. Child
session objects are read-only.

### Turn recording on

#### curl

```bash
curl -X PATCH https://livepeer.com/api/stream/{id}/record \
-H 'content-type: application/json' \
-H 'authorization: Bearer {api-key}' \
-d '{"record":true}'
```

#### JavaScript + axios

```javascript
const axios = require("axios");

axios
  .patch("https://livepeer.com/api/stream/{id}/record", {
    headers: {
      Authorization: "Bearer {api_key}",
      "Content-Type": "application/json",
    },
    data: JSON.stringify({
      record: true,
    }),
  })
  .then((res) => {
    console.log(JSON.stringify(res.data));
  })
  .catch((error) => console.log(error));
```

### Turn recording off

#### curl

```bash
curl -X PATCH https://livepeer.com/api/stream/{id}/record \
-H 'content-type: application/json' \
-H 'authorization: Bearer {api-key}' \
-d '{"record":false}'
```

#### JavaScript + axios

```javascript
const axios = require("axios");

axios
  .patch("https://liveper.com/api/stream/{id}/record", {
    headers: {
      Authorization: "Bearer {api_key}",
      "Content-Type": "application/json",
    },
    data: JSON.stringify({
      record: false,
    }),
  })
  .then((res) => {
    console.log(JSON.stringify(res.data));
  })
  .catch((error) => console.log(error));
```

A `204` no content success status response indicates the `record` value was
updated.
